# Pattern for git commits
"<MODE> <Message>"
## MODE
* REFACTOR - Restructering of content, moving files etc.
* DOC - for changes in documentation (in README or comments)
* DEV - development, to fulfill the task or ading a new feature
* FIX - fixing an error or modifications to feedback

## Message
* MINOR - for typos etc.
* descriptive, but short information what has been changed
